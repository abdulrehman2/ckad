## Question 1. Allow only ingress traffic from api pod to database pod. Also make sure that only the api pod from  namespace `prod` should be allowed 

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
 name: db-policy
spec:
 podSelector:
  matchLabels:
   role: db
 policyTypes:
  - Ingress
 ingress:
  - from:
     - podSelector:
        matchLabels:
         name: api-pod 
       namespaceSelector:
        matchLabels:
         kubernetes.io/metadata.name: prod
    ports:
    - port: 3306   # port of database on which request will land
      protocol: TCP

```


## Question 2 — Allow Traffic from Specific Namespace

Scenario:
Only Pods in namespace frontend-ns should be able to access Pods labeled app=orders in namespace backend-ns.

Answer
```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-frontend-ns
  namespace: backend-ns
spec:
  podSelector:
    matchLabels:
      app: orders
  policyTypes:
  - Ingress
  ingress:
  - from:
    - namespaceSelector:
        matchLabels:
          name: frontend-ns
```

## Question 3 — Allow Egress to External World

Scenario:
Pods labeled app=api in namespace prod should be able to access only Internet/external IPs on port 443 (HTTPS), and no other traffic should be allowed.

```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
 name: external-policy
 namespace: prod
spec:
 podSelector:
  matchLabels:
   app: api
 policyTypes:
  - Egress
 egress:
  - to:
     - ipBlock:
       cidr: 0.0.0.0/32
    ports:
      port: 443
      protocol: TCP
```
