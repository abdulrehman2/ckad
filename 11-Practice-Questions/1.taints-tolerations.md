✅ Question 1 — Taint a Node

Task: Taint node node1 with env=prod:NoSchedule

Answer (kubectl command):

```bash
kubectl taint nodes node1 env=prod:NoSchedule
```
Result:
Pods without toleration for env=prod will not be scheduled on node1.

✅ Question 2 — Remove the Taint

Task: Remove the taint from node1.

Answer (kubectl command):

```bash
kubectl taint nodes node1 env=prod:NoSchedule-
```

Note: - at the end is required to remove the taint.

✅ Question 3 — Pod With Toleration (NoSchedule)

Task: Pod secure-app that tolerates taint security=high:NoSchedule

Answer (Pod YAML):
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: secure-app
spec:
  tolerations:
  - key: "security"
    operator: "Equal"
    value: "high"
    effect: "NoSchedule"
  containers:
  - name: app
    image: nginx
```

Apply it:
```bash
kubectl apply -f secure-app.yaml
```

✅ Question 4 — Pod With NoExecute and Timeout

Task: Pod db-client that tolerates taint dedicated=db:NoExecute for 60 seconds

Answer (Pod YAML):
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: db-client
spec:
  tolerations:
  - key: "dedicated"
    operator: "Equal"
    value: "db"
    effect: "NoExecute"
    tolerationSeconds: 60
  containers:
  - name: client
    image: busybox
    command: ["sleep", "3600"]
```

Apply it:
```bash
kubectl apply -f db-client.yaml
```

Result:
Pod will be evicted 60 seconds after it lands on a tainted node.

✅ Question 5 — Exists Operator

Task: Pod gpu-tester that tolerates any taint with key gpu, ignoring value, effect NoSchedule.

Answer (Pod YAML):
```yaml
apiVersion: v1
kind: Pod
metadata:
  name: gpu-tester
spec:
  tolerations:
  - key: "gpu"
    operator: "Exists"
    effect: "NoSchedule"
  containers:
  - name: app
    image: nginx
```
